

# Design Patterns

设计模式

## Creational Patterns

创建模式

* Singleton 
  * 单例模式
  * 保证一个类仅有一个实例
* Factory Method
  * 工厂模式
  * 让其子类自己决定实例化哪一个工厂类       
* Abstract Factory
  * 抽象工厂模式
  * 创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类
* Builder
  * 建造者模式
  * 使用多个简单的对象一步一步构建成一个复杂的对象
* Prototype
  * 原型模式
  * 创建重复的对象
